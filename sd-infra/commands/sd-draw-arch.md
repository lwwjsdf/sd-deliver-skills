# /sd-draw-arch — 架构图生成

> ⚠️ **执行前确认**
>
> **此 command 会做什么：**
> 根据项目背景生成 draw.io 架构图（逻辑架构、系统流、数据流等）。
>
> **前置条件：**
> - 项目背景已明确（PROJECT.md 存在）
> - 如需整套技术方案，使用 `/design-tech`
>
> **执行步骤概览：**
> 1. 确认架构描述（系统组件、数据流向、部署环境）
> 2. 生成 arch.yaml（架构描述文件）
> 3. 生成 draw.io XML
> 4. 审核调整
>
> 1/y = 确认执行
> 0/n = 取消
> 2/s = 跳过
>
> 每一步执行前我会再次确认。

## 前置条件

- 项目背景已明确（PROJECT.md 存在）
- 如需整套技术方案，使用 `/design-tech`

## 工作流

### Step 1：确认架构描述

从 PROJECT.md 或用户对话中提取架构信息：
- 系统组件和模块
- 数据流向
- 用户类型和访问路径
- 部署环境（云厂商、VPC）

### Step 2：生成 arch.yaml

编写架构描述文件，作为 draw.io 生成的事实层。

### Step 3：生成 draw.io

使用渲染器生成对应的 draw.io XML 文件。

### Step 4：审核调整

根据客户反馈调整架构图细节。

## 图类型选择

| 场景 | 图类型 | 说明 |
|------|--------|------|
| CDP+MAE 整体架构 | 逻辑架构图 | 组件 + 关系 |
| 用户操作路径 | 系统流图 | 分 End User/Biz User/Admin |
| 数据管道 | 数据流图 | 数据流转 |
| 部署拓扑 | 基础设施架构图 | VPC/子网/网络组件 |
| 功能模块 | 功能架构图 | 能力域 |

## 输出

每个图保存为独立的 `.drawio` 文件，可导入 draw.io 编辑。
