## 产品体系

- **CDP**（客户数据平台）= SDH + SA，独立部署
- **MA**（营销自动化）= SF（Sensors Focus），独立部署
- **ETL** 数据接入节点为辅助组件，默认 3 台 8C/32G，按实际数据源的同步频率、历史导入量、清洗复杂度调整

## 部署模式

| 模式 | 节点架构 | 特点 |
|------|---------|------|
| 单机 | 1 节点混部 | 小规模/POC |
| Mini 集群 | 3 节点混部 | 中等规模，最多扩至 5 节点 |
| 标准集群 | 3 元数据 + N 数据节点 | 大规模，支持横向扩容（3+3 / 3+4 / 3+5 / 3+6） |

## 评估输入

| 参数 | 说明 | 默认值/示例 |
|------|------|-------------|
| MAU | 月活跃用户 | 可选 |
| DAU | 日活跃用户 | **优先使用**；未提供时按 MAU × 行业比例估算 |
| DAU/MAU 比例 | 行业比例 | 银行 15%，支付/钱包 30%，高频钱包/超级 App 40% |
| 单用户日事件数 | events per active user per day | 100 |
| 日事件量 | 可覆盖自动估算 | DAU × 单用户日事件数 |
| MA 每天受众上限 | MA 营销受众规模 | 可选 |
| 存储周期 | 数据保留时长 | 默认 1 年 |
| 单亿事件存储 | 35 GB/亿事件 | 可调 |
| 顺序数据修正系数 | 1.1 | 可调 |
| 磁盘使用率上限 | 80% | 可调 |

## 关键公式

### DAU 估算
- 客户提供 DAU → 直接使用
- 未提供 DAU → `DAU = MAU × 行业比例`

### 日事件量
- 客户提供真实日事件量 → 直接使用
- 未提供 → `日事件量 = DAU × 单用户日事件数`

### CDP 顺序盘容量
```
周期内事件量（亿）= 日事件量 × 存储天数 / 1 亿
估算实际使用容量 = 周期内事件量 × 35 GB × 1.1
需配置容量 = 估算实际使用容量 / 0.8
```

### 节点选择逻辑
1. 先按 DAU / 日事件量 / MA 每天受众上限确定基础集群
2. 再按存储周期计算顺序盘目标容量
3. 在云厂商约束（单盘上限、挂载数、IOPS/吞吐）内计算磁盘数量
4. 不足时升级到 3+4 / 3+5 / 3+6 或专项评估

## CDP 配置档次（x86 云主机）

| 配置类型 | DAU 边界 | 日事件量边界 | 节点组成 |
|----------|----------|--------------|----------|
| CDP Mini 标配 | < 100 万 | ≤ 6500 万 | 3 × 混合节点 8C64G |
| CDP Mini 高配 | < 500 万 | ≤ 1.5 亿 | 3 × 混合节点 16C128G |
| CDP 标准 3+3 标配 | < 800 万 | ≤ 4.2 亿 | 3 × 元数据 8C32G + 3 × 数据 16C128G |
| CDP 标准 3+3 高配 | < 1400 万 | ≤ 8.4 亿 | 3 × 元数据 + 3 × 数据（高配） |
| CDP 标准 3+4/3+5/3+6 | 更高 | 更高 | 扩展 Data Node |

> 模板中 1T 单盘是基线值，不是硬性最小值。实际单盘可选 500G/1T/2T/4T 等，取决于云厂商盘型、IOPS、吞吐和实例挂载限制。

## 云厂商磁盘规则要点

| 云厂商 | 顺序盘推荐 | 元数据/随机盘 | 注意事项 |
|--------|-----------|--------------|----------|
| AWS | EBS gp3 | gp3 预配置 ≥5000 IOPS 或 io2 | 单盘 1GiB–64TiB；挂载数受 EC2 实例类型限制 |
| Azure | Premium SSD v2 | Premium SSD v2 / Ultra Disk | 单盘最高 64TiB；VM size 限制远程磁盘数 |
| GCP | Balanced PD / SSD PD | SSD PD | 单盘最高 64TiB；性能受容量+vCPU+实例上限影响 |
| 阿里云国际 | ESSD PL0/PL1 | ESSD PL1 或更高 | 单盘最高 65,536GiB；挂载数受实例规格限制 |
| 华为云国际 | 高 IO / SSD-backed EVS | 超高 IO / 极致型 SSD | 数据盘通常最高 32,768GiB；受实例 Storage QoS 限制 |

## 输出物

- 配置建议表（节点数、CPU、内存、磁盘、云机器类型）
- 内部逻辑说明（不写入客户版）
- Excel / Word / PDF 导出（客户版不包含内部逻辑）

## 边界与专项评估

| 场景 | 处理 |
|------|------|
| DAU > 2200 万 或 日事件量 > 16 亿 | 超出自动评估范围，需专项评估 |
| MA 每天受众上限 > 5 亿 | 超出自动评估范围，需专项评估 |
| 存储在云厂商约束下仍不满足 | 升级 Data Node 或专项评估 |
| 大量历史导入 / 复杂查询 / 高并发任务 | 专项评估 |
