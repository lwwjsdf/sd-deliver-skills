## 产品体系

- **CDP**（客户数据平台）= SDH + SA，独立部署
- **MA**（营销自动化）= SF（Sensors Focus），独立部署

## 部署模式

| 模式 | 节点架构 | 特点 |
|------|---------|------|
| 单机 | 1 节点混部 | 小规模/POC |
| Mini 集群 | 3 节点混部 | 中等规模，最多扩至 5 节点 |
| 标准集群 | 3 元数据 + N 数据节点 | 大规模，支持横向扩容 |

## 关键公式

CDP 顺序盘计算公式（集群环境）：
```
event 存储量（GB）= 日导入量（亿）× 35 × 保留天数 / 0.8
顺序盘总估算 = event 存储量 × 1.1
单数据节点顺序盘容量 = 顺序盘总估算 ÷ 数据节点数
单块顺序盘容量 = 单节点总容量 ÷ 6，向上取整到标准盘型
```

## 评估流程

1. 收集 DAU、日事件量、历史存量、保留天数、增长预期
2. 运行计算器：`python3 scripts/sizing_calc.py ...`
3. 选择部署模式（决策树）
4. 确定配置规格
5. 输出 Excel 模板
