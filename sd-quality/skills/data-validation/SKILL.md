## 核心原则

**必须对比埋点方案文档与实际数据，不允许仅凭肉眼或印象判断。**

## 校验流程

### Phase 1：获取埋点方案

找到当前版本的埋点方案文档，确认版本一致。

### Phase 2：抓取实际数据

从神策系统获取实际上报数据（最近 24 小时）。

### Phase 3：逐项比对

| 检查项 | 通过标准 |
|--------|----------|
| 事件名 | 与方案完全一致 |
| 必填属性 | 全部存在 |
| 属性类型 | 与方案一致 |
| 属性值范围 | 无明显异常值 |
| 触发时机 | 与方案一致 |

### Phase 4：输出差异报告

- 通过：与方案完全一致
- 异常：存在但有问题
- 缺失：方案中有但未上报
