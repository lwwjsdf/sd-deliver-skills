# /sd-run-sit — SIT 执行

> ⚠️ **执行前确认**
>
> **此 command 会做什么：**
> 按 SIT Plan 逐条执行测试用例，记录结果，输出 SIT 测试报告。
>
> **前置条件：**
> - SIT Plan 已设计（`references/sit-plan.md`）
> - 测试环境已就绪
> - 测试数据已准备
>
> **执行步骤概览：**
> 1. 确认测试环境和数据
> 2. 按用例逐条执行（TC-001 ~ TC-XXX）
> 3. 记录实际结果，标记状态
> 4. 缺陷记录和分类
> 5. 输出 SIT 测试报告
>
> 1/y = 确认执行
> 0/n = 取消
> 2/s = 跳过
>
> 每一步执行前我会再次确认。

## 工作流

### Step 1：确认测试环境

- 验证 CDP 连接
- 确认测试账号可用
- 确认数据已导入

### Step 2：执行用例

按 Sit Plan 中的用例逐条执行：
- 自动化部分：通过 OpenAPI 执行 SQL 验证
- 手动部分：生成手动执行步骤，由测试人员操作

### Step 3：记录结果

每个用例标记状态：
- ✅ Pass — 符合预期
- ❌ Fail — 不符合预期
- ⏸️ Block — 因前置问题无法执行
- 🔧 Manual — 需手动执行

### Step 4：缺陷记录

失败用例自动记录：
- 用例 ID
- 失败步骤
- 实际结果 vs 期望结果
- 严重度（P0/P1/P2）
- 建议修复方案

### Step 5：输出报告

```markdown
## SIT 测试报告 — <client>

**执行时间：** YYYY-MM-DD
**测试人：** AI / 交付团队

**汇总**
- 总用例数：X
- 通过：X（X%）
- 失败：X
- 阻塞：X
- 手动待执行：X

**失败详情**
| TC | 场景 | 期望 | 实际 | 严重度 |
|----|------|------|------|--------|
| ... | ... | ... | ... | P0 |

**上线建议**
[ ] 可以上线（所有 P0 通过）
[ ] 不建议上线（存在 P0 失败）
```

## 完成建议

- "SIT 通过？→ 设计 UAT Test Case：`/sd-design-uat`"
- "SIT 失败？→ 修复缺陷后重新执行 `/sd-run-sit`"
